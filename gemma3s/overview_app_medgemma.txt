Overview of 
app_medgemma.py
This is a Gradio-based web application (2588 lines) for interactive medical image/video segmentation using MedSAM2 (Medical Segment Anything Model 2). It's designed for radiologists to review patient MRI scans, segment tumors or other regions of interest, and generate volumetric analysis reports.

🏗️ Architecture
The app uses a multi-process architecture where each user session runs segmentation in a separate multiprocessing.Process, communicating via queues. A background monitor thread cleans up idle sessions after 15 minutes.

Main Thread (Gradio UI)
  ├── Monitor Thread (cleanup idle processes)
  └── Per-User Worker Processes (GPU-intensive SAM2 inference)
       └── Communicate via mp.Queue

8. Gradio Frontend (Lines 1545–2588)
Two-tab interface:

Tab	Purpose
📋 Patient Overview	HTML table of all patients (color-coded: 🔴 red = not reviewed, 🟢 green = reviewed), dropdown to select a patient, auto-loads most recent session's FLAIR scan
🔬 Segmentation	Video viewer + interactive annotation (point/stroke prompts), frame slider, object tracking, NIfTI export with volume report & parcellation analysis
🔄 Typical Workflow
Patient Selection → User picks a patient from the table → App auto-selects the most recent session's FLAIR scan
NIfTI → Video → The 3D NIfTI volume is converted to MP4 (one frame per slice)
Preprocess → SAM2 model is loaded on GPU, video frames are extracted
Annotate → User clicks (point prompt) or draws (box prompt) on the target slice to segment a region
Track → Bidirectional propagation spreads the segmentation across all slices
Export → Masks are converted back to NIfTI, volume is calculated, parcellation analysis maps regions to brain atlas labels
Save → Results are written to patient_results.json and comman_format.json
⚙️ Key Technical Details
Model: SAM2 (Segment Anything Model 2) with configurable config/checkpoint files
GPU: Requires CUDA, uses bfloat16 autocast and TF32 optimizations
Server: Runs on 0.0.0.0:18862 with Gradio 3.38.0
Dependencies: torch, nibabel, ffmpeg, moviepy, cv2, sam2, omegaconf, gradio
